AI Agent层
Agent
Orchestration
🤖 病种规划 Agent
输入病种名称、规划目标等
调用文献检索子Agent → 获取循证证据
输出Disease Skill 草案（JSON）
// Disease Skill Draft JSON { "skillId": "dm2-draft", "disease": "2型糖尿病", "stratLevels": [ { "level": "L1轻度", "criteria": {...} }, { "level": "L2中度", "criteria": {...} } ], "services": [...], "version": "draft-1.0" }
🤖 用户分层规则 Agent
输入Disease Skill中的分层标准
输出分层评分模型（JSON规则引擎）存库，供入组时调用
🤖 用户分层 Agent
输入患者档案JSON（基本信息 + 检测报告）
调用分层规则引擎 → 多维度评分
输出分层结果 + 推荐方案（JSON）
// Stratification Result JSON { "patientId": "P-20260420-001", "diseaseCode": "dm2", "stratLevel": "L2中度", "score": 72, "keyFactors": ["HbA1c=8.2%", "BMI=28"], "suggestedPlanId": "dm2-L2-v1.0" }
JSON写入DB，生成医生审核任务
🤖 执行调度 Agent
输入患者分层 + 绑定方案 + 当前日期
输出任务队列JSON，写入任务表
// Task Queue JSON { "patientId": "P001", "tasks": [ { "type":"用药提醒", "tier":"L1" }, { "type":"血糖上报", "tier":"L1" }, { "type":"随访内容", "tier":"L2" } ] }
🤖 督办 Agent
监听任务过期事件（规则可人工配置）
触发推送提醒 / 上报预警 / 升级处理
🤖 分析优化 Agent
输入全量执行记录 + 患者指标变化数据
输出优化建议 + 新版方案草案（JSON）
// Optimization JSON { "targetGroup": "dm2-L2", "findings": [ "L2用药依从率仅63%", "HbA1c改善率低于预期" ], "suggestions": [...], "newSkillDraft": { "version": "v1.1" } }
JSON写入DB，推送医生审核
